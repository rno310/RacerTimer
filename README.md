# Pin to Win

A sailing race start timer for Apple Watch and iPhone.

Pin to Win runs the same 5- or 3-minute countdown sequences race officers use, with audio and haptic signals at every classic mark, the sync-to-minute and +1 buttons sailors expect, and a count-up race timer once the start gun fires. The watch is the primary device; the phone mirrors it live.

## Features

- **3- or 5-minute countdown**, switchable in settings.
- **Sync** snaps the timer to the current whole minute (the classic sailor's "sync to minute" button).
- **+1** adds a minute when the race is postponed.
- **Reset** at any time from a small icon at the top-left of the watch face.
- **Count-up** after the start gun for elapsed race time.
- **Always-on display** — the timer stays readable even when the watch dims.
- **High-contrast colored background** that changes per minute and **flashes red ↔ white** through the final five seconds.
- **Live phone ↔ watch sync** via WatchConnectivity. The watch stays authoritative if the phone is out of range.

## Signal schedule

Audible tone (880 Hz) plus a haptic at every cue:

| Time | Signal |
|---|---|
| 5:00 | 5 long beeps |
| 4:00 | 4 long beeps |
| 3:00 | 3 long beeps |
| 2:30 | 2 long + 3 short |
| 2:00 | 2 long beeps |
| 1:30 | 1 long + 3 short |
| 1:00 | 1 long beep |
| 0:30 | 3 short beeps |
| 0:20 | 2 short beeps |
| 0:10 | 1 short beep |
| 0:05 → 0:01 | 1 short each second |
| 0:00 | 3-second start tone |

In 3-minute mode the 4:00 and 5:00 marks are not used.

## Privacy

Pin to Win does not collect, transmit, or store any personal data.

- No accounts, no logins.
- No analytics, no tracking, no ads.
- The only network communication is between your iPhone and your paired Apple Watch over Apple's WatchConnectivity framework, used solely to keep the timer state in sync between the two devices on which the app is installed.
- All settings stay on-device.

If this ever changes, this section will be updated and the change will be called out in the App Store release notes.

## Building from source

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`, but the generated `Tutastart.xcodeproj` is committed so the project opens in Xcode without extra tooling.

### Requirements

- macOS with Xcode 26 or newer
- watchOS 26 simulator runtime (download via `xcodebuild -downloadPlatform watchOS` or Xcode → Settings → Components)

### Run

```sh
open Tutastart.xcodeproj
```

In Xcode pick the **Tutastart** scheme and your iPhone (with paired Apple Watch) as the destination, then ⌘R. The watch app is embedded in the iOS app bundle and installs alongside.

To run only the watch app, pick the **Tutastart Watch App** scheme and a watchOS simulator destination.

### Tests

```sh
xcodebuild -project Tutastart.xcodeproj -scheme TutastartTests \
  -destination 'platform=iOS Simulator,name=iPhone 17,OS=26.2' test
```

26 unit tests cover signal schedule generation, timer engine state transitions (start, +1, −1, sync, reset, remote-state apply), and display formatting. They use injected clocks and a recording mock signaler — no real time elapses.

### Regenerate the Xcode project

After editing `project.yml`:

```sh
brew install xcodegen   # one-time
xcodegen generate
```
